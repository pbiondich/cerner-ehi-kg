# EEM_NOTIFICATION_DETAIL

> The EEM_NOTIFICATION_DETAIL table stores data for admission notification transactions. Will be used by clients to create work lists or reports on admission notifications.

**Description:** EEM Notification Detail  
**Table type:** ACTIVITY  
**Primary key:** _(not published — see note)_  
**Columns:** 12

[← index](../index.md)

## Columns

| # | Column | Type | Null? | Flags | Definition |
|--:|--------|------|:-----:|-------|------------|
| 1 | `EEM_NOTIFICATION_DETAIL_ID` | DOUBLE | NOT NULL |  | The unique primary key of this table. It is internally generated by Cerner. |
| 2 | `ENCNTR_ID` | DOUBLE | NOT NULL | FK→ | Identifier of the encounter associated to the admission. |
| 3 | `HEALTH_PLAN_ID` | DOUBLE | NOT NULL | FK→ | Identifier of the patient's health plan |
| 4 | `INTERCHANGE_ID` | DOUBLE | NOT NULL |  | Interchange ID. Refers to interchange_id on the EEM_TRANSACTION table. It is the Primary identifier for a transaction but is not the PK of the EEM_TRANSACTION table. |
| 5 | `PAYER_ID` | DOUBLE | NOT NULL | FK→ | The identifier of the organization that is the payer of the transaction. Also known as the carrier organization. |
| 6 | `REPLY_DT_TM` | DATETIME |  |  | The date and time the transaction was returned. |
| 7 | `SENT_DT_TM` | DATETIME | NOT NULL |  | The date and time the transaction was sent outbound. |
| 8 | `UPDT_APPLCTX` | DOUBLE | NOT NULL |  | The application context number from the record info block. |
| 9 | `UPDT_CNT` | DOUBLE | NOT NULL |  | Set to 0 on insert. Incremented by 1 on update. Used to recognize update conflict where data in a row updated by one application is at risk of being lost by a second application attempting to update the row. |
| 10 | `UPDT_DT_TM` | DATETIME | NOT NULL |  | The date and time the row was last inserted or updated. |
| 11 | `UPDT_ID` | DOUBLE | NOT NULL |  | The person_id of the person from the personnel table (prsnl) that caused the last insert or update of the row in the table. |
| 12 | `UPDT_TASK` | DOUBLE | NOT NULL |  | The registered (assigned) task number for the process that inserted or updated the row. |
