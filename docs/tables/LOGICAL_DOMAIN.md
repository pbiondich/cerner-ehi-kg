# LOGICAL_DOMAIN

> The logical_domain table is used to represent a logical database partition used to sub-divide data on tables where data is not shared between partition owners. An example would be having multiple clients using the same domain and Millennium tables. Each client would have their own logical domain identifier.

**Description:** Logical Domain  
**Table type:** REFERENCE  
**Primary key:** `LOGICAL_DOMAIN_ID`  
**Columns:** 16
